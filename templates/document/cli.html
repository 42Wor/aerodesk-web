AeroDesk CLI Reference Guide

Complete operational manual for the compiled `aerodesk` executable. Learn how the local cache engine, multi-monitor configuration wizards, and system fallbacks work on Unix and Windows systems.

90-Day Asset Lifespan Rule

AeroDesk saves backgrounds locally to prevent redundant downloads and save your bandwidth. Cached assets expire automatically **90 days** from their last execution date unless accessed again.

Linux/macOS Cache: ~/.local/share/backgrounds/aerodesk/
Windows Cache: %AppData%\Roaming\AeroDesk\

Multi-Monitor Layout Architecture

Under Linux (Hyprland), AeroDesk queries your active displays and compiles distinct rendering daemons in parallel. Configured displays receive targeted video rendering threads using `mpvpaper` to maintain low system latency.

Auto-Detection: Queries JSON metadata via hyprctl monitors -j.
Process Separation: Independent process loops avoid global thread blocking.

Config Parameters & Flags

      

        In addition to the interactive configuration menu, the binary accepts command line arguments to update settings programmatically.
      


      

        

          
            

              Argument Flag
              Accepts Value
              Operational Definition
            

          
          
            

              --monitor
              all | default | list
              Specifies displays (e.g., --monitor="eDP-1,DP-1").
            

            

              --mpv-opts
              string
              Injects custom player initialization parameters directly to the engine render loop.
            

            

              --all
              boolean (flag)
              Enforces dynamic, immediate file purging of the whole cache when passed to clean.
