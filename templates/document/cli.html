AeroDesk CLI Reference Guide

Complete manual for the compiled `aerodesk` executable. Learn how the local cache engine, Hyprland autostart linkages, and system-level rollbacks work across different operating systems.

Local Caching & 90-Day Lifespan Rule

AeroDesk downloads selected live backgrounds locally to reduce network overhead and prevent repetitive downloads. Each downloaded file has an explicit **90-day retention lifespan** from its last used/modified timestamp.

Linux/macOS Cache Path ~/.local/share/backgrounds/aerodesk/
Windows Cache Path %AppData%\Roaming\AeroDesk\
Asset Expiration Check Every 'apply' command triggers age evaluation.

CLI Commands Reference

aerodesk list

Query

Queries your public Vercel registry (`wallpapers.json`) and prints a tabular overview of all available wallpapers, their format, and their local cache status.

Terminal Execution 
$ aerodesk list
ID         TITLE                                    FORMAT     STATUS
4fa0a7d    Furina - 4K Cook Pardon!                 MP4        Cached (12d old)
d16a978    Summer Strawberries                      MP4        Not Cached

aerodesk apply <id>

Deployment

Downloads the wallpaper asset mapped to the given hash ID, saves it, creates a dynamic symlink (`current_wallpaper`), and instantly updates the background using platform-specific engines.

Terminal Execution 
$ aerodesk apply 4fa0a7d
[+] Valid local asset found (12 days old). Skipping network download.
[+] AeroDesk symlink updated.
[✔] AeroDesk live background applied.

aerodesk apply 0

System Rollback

Safely deactivates AeroDesk. It terminates the rendering processes (`killall mpvpaper`), removes the autostart configurations from your environment files, and restores your system back to your original configuration.

Terminal Execution 
$ aerodesk apply 0
[*] Initiating fallback sequence. Disabling AeroDesk workspace engine...
[✔] Original hyprland.conf configuration restored from backup.
[✔] AeroDesk successfully deactivated. Original system background is restored!

aerodesk clean [--all]

Cache Maintenance

Scans the local storage directory and purges files older than 90 days. Using the optional flag `--all` force-deletes all cached resources inside your AeroDesk directory immediately.

Terminal Execution 
$ aerodesk clean
[*] Scanning local workspace storage path...
[*] Removed expired resource: Alice Montage.mp4 (93 days old)
[✔] Cache cleanup complete. Removed 1 files from cache.
